# Curated Shortlist

## B01. Double Diamond

- source: Design Council
- keep:
  - separate problem framing from solution convergence
  - make divergence and convergence visible in artifacts
- implication for `bagakit-brainstorm`:
  - the best default use is for option framing and narrowing, not endless idea generation

## B02. Nominal Group Technique

- source: ASQ practice guide
- keep:
  - generate ideas independently first
  - score independently before group merge
- implication for `bagakit-brainstorm`:
  - `expert_forum` should support independent position rows before the final discussion merge
  - this is a good antidote to dominant-voice drift

## B03. Creative Problem Solving

- source: Creative Education Foundation
- keep:
  - explicit clarify / ideate / develop / implement rhythm
  - each phase should leave a concrete output
- implication for `bagakit-brainstorm`:
  - current stage artifacts are directionally right
  - the default path should stay understandable without forcing every advanced option

## B04. Delphi Method

- source: Delphi method literature
- keep:
  - asynchronous convergence under uncertainty
  - structured round-by-round refinement
- implication for `bagakit-brainstorm`:
  - deep expert-forum modes make sense as an escalation path, not as the mandatory default

## L01. Current bagakit-brainstorm runtime

- source: sibling standalone skill
- keep:
  - explicit artifact stages
  - archive and handoff gate
  - method-pack awareness
- implication for canonical absorption:
  - Bagakit should keep the bounded planning workflow, but simplify the default mode and reduce name-bound integrations

## Short Conclusion

The strongest canonical Bagakit brainstorm contract is:

- structured artifacts
- explicit decision/handoff outputs
- method selection when needed
- deeper expert review only when uncertainty or disagreement warrants it

The weakest current pattern is treating the heaviest expert-forum path as the
implied baseline for every brainstorm run.
