# 平衡说明

这个样本代表一篇短技术说明。它有足够的连接句，因此不会把读者推进连续清单。

## 背景

文档先说明读者面对的问题。它先讲清楚不变量，再引入少量检查动作。

## 决策

实现把协议权威和展示状态分开。这样每个状态变化都有单一来源，派生视图也可以从同一来源重建。

两个小检查仍然有用。

- 确认可见角色
- 确认记录位置

这些检查刻意保持很短。上下文由段落承担，列表只起到辅助作用。

```text
- this generated log line is not prose
- this generated log line is not prose either
```

## 结果

读者现在能看到变化、原因和证据。评审者仍然可以不同意设计，但不需要从连续列表里重建论证。
