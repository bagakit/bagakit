# Synthetic Operator Guide

This synthetic sample is not copied from a real project. It models a guide that
looks tidy in a hard lint pass but still asks the reader to process too many
separate checklist cues before the connective story has formed.

## Start From The Current Run

The operator begins by reading the current run as a single stateful handoff.
That handoff should explain what is happening now and why the next action is
worth taking, instead of only naming objects.

- confirm the active item
- check the current owner
- inspect the latest receipt
- compare the visible surface
- decide whether the run is blocked
- record the reason

The next paragraph gives the reader a small amount of orientation. It explains
that these checks are useful, but it does not yet turn them into a complete
story about what the system guarantees.

- read the command result
- read the status summary
- read the open decision
- read the available handoff
- read the last verification

After two list blocks, the guide resumes prose. A human reviewer would likely
ask for the lists to become a short narrative with one compact table.

## Continue The Loop

The loop should make progress visible before it adds more branches. Readers need
to understand which state is authoritative and which state is only presentation.

- choose the next transition
- run the smallest check
- preserve the evidence
- update the task receipt
- keep the active owner clear

This prose line tries to bridge the action list back to the invariant, but the
section still depends on repeated scanning.

- verify the current state
- verify the expected command
- verify the observable output
- verify the rollback condition

The repeated pattern is intentional. It gives the regression test enough shape
to detect medium list clusters without needing any project-specific terms.

## Hand Off With Evidence

The handoff should compress what changed and why the next reader can trust it.
That is the most important sentence in this synthetic sample.

- summarize the completed transition
- link the verification receipt
- name the unresolved risk
- name the next action
- keep private data out of examples
- avoid local machine paths

Evidence needs to be durable and small. The sample ends with a balanced sentence
so the file is not only a list fixture.

motivated the advisory metrics. The document should stay below the older whole

The added prose is generic on purpose. It does not describe a real repository,

miss.

The reviewer signal should therefore come from local density and repetition, not
from a globally extreme list percentage. That distinction matters because many
technical notes contain valid command lists, but only some of them let the lists
replace the connective argument.
