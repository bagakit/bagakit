## Bagakit Takeaways

For high-craft multi-section webpage work, generated design references should
often be a section frame set instead of one tall page image. The useful target
is implementation-readable evidence: each frame should expose layout,
hierarchy, spacing, typography scale, image treatment, component language, and
the section's conversion or comprehension job.

Use `section-reference-plan.md` when a generated landing, marketing, product,
portfolio, or full-site reference has multiple sections. It should record the
section count, ordered section names, section job, frame label, canvas ratio,
frame anchor, background or material treatment, image role, CTA role, density
level, and planned responsive implication.

Use `section-frame-continuity-ledger.md` when a reference set is split across
multiple generated frames. It should preserve one brand world across palette,
type scale, spacing cadence, CTA family, radius and material language, imagery
grade, icon or mark style, and copy tone. Variation is welcome; identity drift
is not.

For longer pages, the plan should avoid repeating one obvious composition
pattern. Mix frame anchors, density, image-to-text balance, and treatment
intensity across the scroll so the page has rhythm instead of cloned slabs.
The first viewport should not default to the classic text-plus-visual split
unless the brief or reference survey makes that the strongest choice.

Image usage should be structural. Product crops, editorial photos, full-bleed
visuals, material textures, or generated asset fragments can carry hierarchy
and brand mood. Tiny decorative thumbnails, fake dashboards, and unrelated
stock-like imagery should be rejected.

One visual throughline may connect the frames. One delayed-recognition detail
may appear once if it improves scan order or recall. Repeated gimmicks and
theme swaps weaken the set.

The review gate should reject:

- a multi-section reference collapsed into one tall collage when section
  frames were required
- fewer generated frames than planned sections
- disconnected frames that no longer read as one brand system
- repeated frame anchors that make the page feel template-generated
- image-free or card-only output when the brief calls for visual direction
