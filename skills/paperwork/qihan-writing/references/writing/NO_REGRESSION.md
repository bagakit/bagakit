# No Regression Rule

当用户反馈“结构/口吻/风格”时，重写很容易误伤：为了顺滑和结构，会把原本有价值的材料（参考文章、技能调研、评测标准、脚本设计细节）删掉。

## 规则

1) **默认不删信息**：除非用户明确要求精简，否则改写应以“合并/改述/重排/追加”为主。
2) **四类信息块必须保留**（如果原文存在）：
   - 写作指导来源（文章 / 链接 / 要点）
   - 核心判断依赖的证据或样本口径
   - 评测标准（结构 / 零散度 / 排版比例等）
   - 实施细节（检查项、阈值、流程或下一步）
3) **改写前做 inventory**：先列出原文有哪些信息块；改写后逐项对照，确保都还在。
