# Inline Code Usage

对外文章默认少用 inline code。

很多文档渲染工具对反引号包裹的词渲染很差。普通概念、英文短语、文章命题、修辞性标签，用加粗、直接改写，或手动强调，通常都比反引号更清楚。

## 什么时候可以用

只有在这几类情况，才优先考虑 inline code：

- 真的是代码
- 命令、flag、路径、字段名、schema key
- 一个名字的“代码身份”本身就是论证对象

## 什么时候不要用

- 普通英文概念词
- 修辞性标签
- 文章主命题
- 只是为了“看起来像技术文章”而加上的英文词

## 优先级

1. 直接改写成普通正文
2. 用加粗强调
3. 只有代码身份真的重要时，才用 inline code
