# Writing Scenario Router

写作很适合场景化，因为很多失败不是“不会写”，而是“用错了写法”。

同样是技术写作：

- 从零起草
- 研究综述
- 代入用户视角重写
- 句子级去 AI 味
- 洞察问答后回写
- 终稿收束

它们面对的对象、风险和工作流都不一样。

所以，进入 `qihan-writing` 后，第一件事不该是直接开写，而应该是先选场景。

但场景还不等于正文骨架。场景定完以后，还要再用
`references/writing/NARRATIVE_ANGLE_SELECTION.md` 选一个主叙事视角。

## 场景总表

| 场景 | 触发信号 | 目标 | 最常见失败 |
| --- | --- | --- | --- |
| `S1_from_scratch` | 只有主题/碎片材料，还没有成文 | 从零搭出完整文章 | 上来就堆材料，主轴发虚 |
| `S2_synthesis` | 材料很多，需要压主线和方法 | 从材料里长出一条判断线 | 写成材料串讲 |
| `S3_user_voice` | 用户要求“像我本人写” | 贴近用户判断和口吻 | 写成成熟编辑代写稿 |
| `S4_insight_loop` | 用户想法比草稿深，但还没完全说出来 | 先问答再压主轴 | 问题弱、默认答案弱、用户懒得答 |
| `S5_rewrite` | 已有草稿，需要句子级/段落级改写 | 去 AI 味、提强度、纠正叙事位置 | 只改命中句，不回扫全文 |
| `S6_final_polish` | 主稿已成形，只差最后一轮 | 收束节奏、去多余、提力量 | 过度润色，反而把判断写轻 |

## 如何选择

### 选 `S1_from_scratch`

当输入主要是：

- 零散笔记
- 一组链接
- 粗提纲
- 很强的主题，但还没成文

优先目标：

- 先把主轴、结构和几个核心判断搭出来

### 选 `S2_synthesis`

当输入主要是：

- 多份分享 / 多篇笔记 / 多场会议
- 需要“总结出规律”
- 需要同时兼顾“可读”和“可审计”

优先目标：

- 不要再加材料，先压“共性 / 演化 / 方向”
- 把正文和附录的分工提前想清楚

### 选 `S3_user_voice`

当用户明确说：

- 从我角度写
- 模仿我的风格
- 不要像 agent / 编辑

优先目标：

- 作者在场感
- 判断归属
- 风格禁区

### 选 `S4_insight_loop`

当出现下面任一信号：

- 用户想得比稿子深
- 用户不断说“还是浅”
- 用户开始用改句子的方式暴露世界观

优先目标：

- 先跑问答环，再回写正文

但如果“还是浅”指的是：

- 缺时代背景
- 缺理论基础
- 缺证据或样本边界
- 现有材料根本撑不起主命题

就不要选 `S4_insight_loop`，先去 `references/workflow/DEPTH_ESCALATION_LOOP.md`。

### 选 `S5_rewrite`

当已有一版成文，但用户不断给：

- 原句 -> 目标句
- “这句不对”
- “这种说法 AI 味太重”

优先目标：

- 不只改一句，要回扫同类问题

### 选 `S6_final_polish`

当主稿已经对了，问题只剩：

- 节奏不稳
- 个别长句拖
- 个别标题不够好

优先目标：

- 保持判断不丢
- 去掉最后一层解释腔和多余劲儿

## 路由原则

1. 一个任务可以跨多个场景，但每一轮最好有一个主场景。
2. 如果用户开始连续重写句子，优先从 `S2` 或 `S3` 切到 `S5`。
3. 如果用户不断暴露出更深世界观，优先切到 `S4`。
4. 如果“浅”的来源是基础层缺口，不要误切到 `S4`，先进入 depth escalation。
5. 如果已经接近完稿，不要回到 `S1` 或 `S2` 的大拆大建。

## 最容易犯的错

1. `S2` 的任务，写成 `S1`
   - 结果：补材料补个没完，主轴越来越慢
2. `S3` 的任务，写成 `S2`
   - 结果：内容对了，但不像用户本人
3. `S5` 的任务，写成局部修补
   - 结果：用户不停追着问“全文也改了吗”
4. `S6` 的任务，重新大拆
   - 结果：把已经成形的判断写散了

## 推荐用法

进入写作任务后，先问自己三句：

1. 现在最主要的问题，是缺结构，缺主轴，缺风格，还是缺最后一轮收束？
2. 用户是在补材料，还是在暴露判断？
3. 这一轮结束后，希望稿子发生哪一种变化？

答完再选场景。

场景选完，再选视角。

如果场景选完以后，route memo 仍然写不稳，先不要硬选视角，直接转去 `references/workflow/DEPTH_ESCALATION_LOOP.md`。
