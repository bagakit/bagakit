# First-Person Point Of View

Use this reference only when the user explicitly asks for first-person writing
or when the deliverable's genre requires a named authorial perspective.

## Default Meaning

First person means the artifact speaks as the accountable author of the work.
It does not grant permission to invent private facts, feelings, credentials, or
preferences.

When source material does not prove a first-person claim, mark it as a judgment,
memory, hypothesis, or open gap rather than presenting it as fact.

## Checklist

- Does the user or route memo explicitly authorize first person?
- Is "I" used for accountability, experience, judgment ownership, or a decision
  boundary?
- Can neutral subjects such as "this system", "the draft", "the evidence", or
  "the current version" carry the sentence more cleanly?
- Are unsupported biographical details, private motivations, or personal taste
  being invented?
- Does the first-person voice preserve evidence boundaries and uncertainty?
- Does the draft avoid referring to the author as "the user" or by an unrelated
  personal name?

## Rewrite Rule

When first person is unnecessary, move the sentence back to the object:

- Weak: "I think this design is risky."
- Better: "This design is risky because the rollback path depends on manual
  state reconstruction."

Keep first person only when ownership is part of the claim:

- "I chose the smaller rollout because the failure signal is still ambiguous."
