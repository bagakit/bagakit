# Bagakit Codex Webpage Design

Design-led webpage implementation skill for Codex.

It binds requirements discussion, image2 design reference generation, frontend
implementation, browser debugging, and visual parity iteration into one SWE
workflow.

The workflow is reference-context-first: image2 is the default reference
generator when no stronger design reference exists, but Figma, screenshots,
approved mockups, real assets, design systems, or structured content can be
stronger references. Do not implement from text requirements with no design
reference.

For spatial or stateful pages, machine validation is not enough. The workflow
also gates canvas safe zones, mobile touchable states, artifact freshness, and
independent visual judge aggregation before claiming parity.
