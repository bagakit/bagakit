# Image2-Only Top-Down Sprite Asset Contract Template

Copy this into the isolated workspace as `asset-contract.md` and edit names/counts only when the consuming runtime contract changes.

## Non-Negotiable Constraint

The visual source art must be generated by image2/image generation. Procedural drawing may only be used for:

- alpha removal
- cutting source strips into frames
- foreground crop
- normalization
- validation
- preview contact sheets

Procedural character drawing or frame fabrication is a failure.

## Goal

Create a complete top-down 2D character asset package for a survivors-like game prototype.

Style target:

- stylized pixel-action sprite sheet
- top-down 3/4 camera, not side-view
- strong silhouette
- chunky cel-shaded color planes
- limited palette
- game-scale readability

## Required Source Images

Create image2 source images under `source/`:

- `model-sheet-source.png`
- `idle-front-source.png`
- `idle-back-source.png`
- `walk-front-source.png`
- `walk-back-source.png`
- `shoot-front-source.png`
- `shoot-back-source.png`
- `hit-front-source.png`
- `hit-back-source.png`

Sources may use flat chroma-key green. Final runtime sheets must be RGBA alpha PNG.

## Required Runtime Sheets

Frame size: `128x128`.

- `final/hero-image2-idle-front.png`: 6 frames, `768x128`
- `final/hero-image2-idle-back.png`: 6 frames, `768x128`
- `final/hero-image2-walk-front.png`: 8 frames, `1024x128`
- `final/hero-image2-walk-back.png`: 8 frames, `1024x128`
- `final/hero-image2-shoot-front.png`: 6 frames, `768x128`
- `final/hero-image2-shoot-back.png`: 6 frames, `768x128`
- `final/hero-image2-hit-front.png`: 4 frames, `512x128`
- `final/hero-image2-hit-back.png`: 4 frames, `512x128`

Also deliver:

- `preview-contact-sheet.png`
- `validation-report.json`
- `independent-image2-validation-report.json`
- `visual-metrics-report.json`
- `review-disposition.md`
- `generation-log.md`
- `README.md`

## Review Disposition

Create `review-disposition.md` after technical validation and visual review.
Use this compact shape:

```markdown
verdict: pass
reviewer: <role-or-name>
accepted_warnings: none
rejected_warnings: none
runtime_contract_deviations: none
notes: <short evidence-bound rationale>
```

Use `verdict: conditional` only when the consuming runtime owner accepts the
remaining issue. Use `verdict: fail` when regeneration is required before
integration.

## Hard Failure Conditions

Reject if:

- source art is not image2-generated
- final sheets are procedural character drawings
- any final PNG has green background leakage
- any frame is empty or effect-only
- front/back are just flips
- walk has no readable gait cycle
- standing shoot shows walking legs
- side-view/platformer pose dominates
- identity changes between actions
- required file/frame dimensions are wrong
