# Brainstorm Method Playbook

## Goal

Provide a compact method pack so each brainstorm run can choose a fit-for-purpose facilitation model instead of using one generic discussion style.

## Method Cards

### 1) Double Diamond (Design Council)

- Source: Design Council Double Diamond.
- Use when:
  - problem framing is still fuzzy,
  - team keeps jumping to solutions too early.
- Apply in brainstorm:
  - Discover/Define -> `input_and_qa` + `finding_and_analyze` problem framing.
  - Develop/Deliver -> option convergence + `outcome_and_handoff`.
- Expected artifact signal:
  - explicit divergence list and convergence rationale.

### 2) Nominal Group Technique (ASQ)

- Source: ASQ Nominal Group Technique.
- Use when:
  - dominant voices are suppressing others,
  - you need equal participation and ranked priorities.
- Apply in brainstorm:
  - silent idea generation per expert,
  - round-robin sharing,
  - independent scoring before discussion merge.
- Expected artifact signal:
  - traceable score table + ranked shortlist.

### 3) Creative Problem Solving (Creative Education Foundation)

- Source: CEF CPS process (Clarify / Ideate / Develop / Implement).
- Use when:
  - team is stuck between idea quantity and execution feasibility.
- Apply in brainstorm:
  - map CPS phases to workflow stages and keep each phase output explicit.
- Expected artifact signal:
  - each stage has a concrete output and next action.

### 4) Delphi (Dalkey & Helmer)

- Source: Dalkey & Helmer, "An Experimental Application of the Delphi Method..."
- Use when:
  - expert consensus is needed under uncertainty,
  - synchronous debate causes anchoring or status pressure.
- Apply in brainstorm:
  - run asynchronous rounds,
  - feedback summary between rounds,
  - converge on explicit consensus/disagreement items.
- Expected artifact signal:
  - round-by-round convergence notes and unresolved assumptions.

## Evidence Notes

- Study signal: nominal (independent) groups can outperform interacting groups on idea quantity and quality in many settings; production blocking remains a key risk in live group brainstorming.
- Use this as a guardrail:
  - add individual ideation before group debate,
  - keep scoring independent before final merge.

## Method Selection Guide

Pick one primary method and optional secondary enhancer per run:

- If scope is unclear -> primary `Double Diamond`.
- If participation quality is the risk -> primary `NGT`, optional `Delphi`.
- If creativity vs feasibility balance is weak -> primary `CPS`, optional `NGT`.
- If consensus quality is the risk -> primary `Delphi`, optional `Double Diamond`.
