# Bagakit Design Core

Medium-neutral design reasoning for Bagakit.

This skill absorbs two kinds of practice into Bagakit terms:

- brand tonality should become concrete design axes, not loose adjectives
- design review should cover the whole surface with evidence, not taste alone

It is meant to compose with implementation skills such as
`bagakit-codex-webpage-design`, Figma workflows, and image-generation workflows.
The implementation skill owns rendering, screenshots, code, and delivery;
design core owns the design packet and checkpoint reviews.

## Core Surfaces

- `SKILL.md`
  - thin operating protocol
- `references/design-core-contract.toml`
  - structured stage, artifact, and guard contract
- `references/artifact-contract.md`
  - runtime artifact names and packet shape
- `references/brand-tonality.md`
  - evidence-to-tone-axis method
- `references/design-rule-system.md`
  - review dimensions and rule-governance method
- `.bagakit/design/`
  - project-local synthesis, task packets, and review outputs when
    materialized
