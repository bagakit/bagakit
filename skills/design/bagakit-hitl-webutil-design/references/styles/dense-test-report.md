# Dense Test Report

## Route

- Compact summary header, grouped rows, matrix or table-heavy scan pattern.

## Good For

- Reviewing many outcomes, blocker distribution, or final QA state.

## Control Treatment

- Favor scan-first summaries over step-by-step execution panes.

## Failure Signals

- The page looks like an interactive console but lacks execution affordances.
- Summary density buries blocker visibility.
